## 4. REFACTOR RISK ASSESSMENT

**Overall risk: VERY HIGH**

| Issue | Detail |
|-------|--------|
| Globals everywhere | ~40+ globals mutated across 70 functions. Removing any single global requires touching every engine. Cannot be done incrementally per-engine. |
| Import-time side effects | argparse runs on import, HTTPS request to GitHub for online JSON config, HTTPS country lookup, os.environ mutations, signal handlers, atexit.register, conditional driver imports — all execute at module import. Importing from a test triggers network calls. |
| Runtime mutation of dispatch state | `set_translation_function()` writes a global function pointer; called with two different signatures depending on `engine_method`. |
| In-flight engine_method rewrites | `main()` flips `engine_method` from `phrasesblock` to `singlephrase` if DeepL fails, calls `set_translation_function` again, closes driver, recreates it. Locking down `engine_method` breaks this fallback. |
| Dead/buggy branches | Two consecutive `elif translation_engine == 'deepl'` branches (L4749, L4754) — second is unreachable. Three identical `import os, re, time, shutil` blocks at L7647–7667. |
| ⚠️ Security | Hard-coded Perplexity API key at L4648 — must be moved to env var. |
| Aligner inline in save_docx_file | `_simple_split_docx` + `FASubtitleAligner` buried inside file-save loop. |
| PROGRESS markers in two paths | Single-call: 15/30/65/75/90/100. Block-loop: 25/50/75 proportional. Scattered across codebase. `local_launcher.py` depends on exact stdout sequence. |
| DOCX parallel arrays | Index off-by-one (i+1 reader-style vs row_n writer-style). Any refactor to dataclass must preserve every existing index. |
| Engine-specific driver type frozen at import | Switching engines at runtime is impossible. |
