<MANDATORY_NORMALIZATION>
Hygiene transforms. Apply on ALL lines in PHASE_0_BASE (incl. flagged CLEAN, SHORT_LINE, FRAGMENT). Required transforms, not byte-id.
MN-1 QUOTE_STYLE — " " only. «» → " ". Terminal punct outside quote. Nested → ASCII '.
MN-2 EZAFE_FORM — silent ه → هٔ (خانهٔ، درجهٔ). Never ه‌ی. ZWNJ for: prefixes (می‌، نمی‌)، suffixes (ها، تر، ترین)، established compounds (هم‌خانه، هم‌ساکن، دست‌وپاگیر، به‌شمار، بی‌خانمان). Avoid decorative ZWNJ, especially post-preposition.
MN-3 HARAKAT — absent in [FA] → absent in output. Adding forbidden except homograph; ceiling/100 lines: NEWS/FACILITATOR ≤3, SAGE ≤5. Never on SMTV/proper names.
MN-4 NO_BAASHE — "باشه"/"باشهٔ" → "بله" / "درست است" / "حتماً". Mandatory broadcast lift. Applies inside ordinary quotes too.
MN-5 NO_TOOSAT — "توسط" treatment:
       PREDICATE line → active voice within same line; preserve passive only if agent unknown/absent.
       FRAGMENT line → no active conversion possible; replace "توسط" with prepositional by context (از سوی / به دست / زیر نظر / با حمایت).
       Never invent agent.
MN-6 NO_SEMICOLON — "؛" OUTSIDE quote spans → "،" or clause split within SAME line. Inside VERBATIM/LITERARY quote (LS-10) → byte-id; inside ordinary quote → normalize per this rule.
MN-7 SERIAL — قسمت N از M. قسمت locked.
MN-8 LATIN_LEAKAGE — outside W3/W4 → Persianise (AI→هوش مصنوعی | EU→اتحادیه اروپا | DNA→دی‌ان‌ای). Applies inside ordinary quotes.
</MANDATORY_NORMALIZATION>

On MISMATCH:

</SEMANTIC_ANCHORS>

<CONSERVATISM_GATE>
Run BEFORE every STYLISTIC edit. Trigger → ABANDON, keep BASE for this line. (BASE, not raw [FA] — hygiene + anchor fixes preserved.)
MANDATORY_NORMALIZATION already applied in PHASE_0_BASE — does not pass through this gate.

G1 [CRITICAL] MEANING — semantic shift ≥5% → ABANDON. Negation/quantifier/speaker/cause-effect/tense never sacrificed.
G2 [STRONG] DOUBT — "native speaker clearly prefers edited" → proceed. "Both acceptable" → ABANDON.
G3 [STRONG] LENGTH — broadcast CPL budget. ABANDON if BOTH: edited >42 CPL AND edited >120% of original CHAR count. Word-count expansion alone is NOT a fail.
G4 [STRONG] FRAGMENT_ORDER — fragment + issue is clause-level word order → ABANDON.
G5 [STRONG] SHORT_LINE — 1–3 content words → skip STYLISTIC rewrites only. PHASE_0_BASE has already applied hygiene + anchors.
G6 [STRONG] SAGE_WARMTH — SAGE block, warm dignified phrasing → CLEAN unless genuinely broken.
G7 [STRONG] MEANING_PRESERVATION — Lexical substitution PERMITTED if it preserves same meaning + speaker + polarity + quantity + tense within same line. Required semantic content (predicate, entity, negation, quantifier, cause-effect) must remain.
G8 [CRITICAL] BOUNDARY — content word (original or substituted) moved to different line → ABANDON.
G9 [STRONG] HANDOFF_TRUST — BASE reads native (no MT residue, no EN clause shadow, no calque) → CLEAN.

Calibration: 15–25% diagnostic range. Not a floor, not a cap.
</CONSERVATISM_GATE>
