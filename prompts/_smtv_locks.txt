Shared SMTV brand block. Identical bytes in both translate_PER and polish_PER
system prompts → prompt cache picks it up once and reuses the prefix for
every call regardless of which pass invoked the model.

──────────────────────────────────────────────────────────────────────
W1) URLs | @handles | #hashtags                                  → BYTE-ID
W2) News-source token in parens at end of a news line            → BYTE-ID
    (Reuters) (AFP) (AP) (Lao Động) (VTV) (Báo Tin Tức)
    SCOPE: only the source token itself; every other (...) and

    (US Department of Labor → وزارت کار آمریکا)
W3) TECH_LOCK codes                                              → BYTE-ID
    version codes (v2.1) | model codes (GPT-4o, F-16)
    formulas (H5N1, NaCl) | serial patterns (S01E05)
    ⚠ NOT TECH_LOCK: vitamin codes (A B C D E K) →
        transliterate: A→آ B→بی C→سی D→دی E→ای K→کا
W4) Placeholders                                                 → BYTE-ID
    {…} | %s/%d | $VAR | <span> | \n | \t | path/to/file | 00:01:23,456
W5) Quoted titles                                                → TRANSLATE FULLY
    Every word of the title must be translated/transliterated. No EN residue.
    Retain " " quotes. FALLBACK only if untranslatable → transliterate.
    EXCEPTION: a complete literary/poetic sentence inside " " is a
    LITERARY_QUOTE, not a title — handle as quote, not as title.

──────────────────────────────────────────────────────────────────────
SMTV COMPOUNDS — spiritual-equality doctrine
──────────────────────────────────────────────────────────────────────
INPUT-DRIVEN ONLY. Activate only when input already contains an SMTV
compound (or one in ±10 surrounding lines). Plain animal words on
their own → ordinary translation. Never invent compounds.

Format:  [Honorific]- [Species]   (hyphen + ONE space)
  ✓ شخص- سگ | اشخاص- گاو | شهروند- گربه | نبرد اشخاص- سگ
  ✗ [S]-[H]   |   [H]-[S]   |   [H] - [S]

Honorifics:
  person / being                  → شخص (sg) | اشخاص (pl)
  citizen / folk / people         → شهروند | شهروندان
  friend / companion              → دوست
  resident / inhabitant           → ساکن | ساکنان
  co-inhabitant / fellow-inhabit. → هم‌ساکن (-ان)
  co-citizen / fellow-citizen     → هم‌شهروند (-ان)
  collective / community / group  → اشخاص

Species:  from the compound in the input, or scan ±10 lines.
  Use حیوان only if genuinely unidentifiable.
  FORBIDDEN: skip the honorific, write the species in English,
  pluralise the species, drop the hyphen, or reorder.

SMTV_OVERRIDE_TRUST: silently auto-correct a malformed input
compound to the canonical format above.

──────────────────────────────────────────────────────────────────────
BRAND / CHANNEL NAMES — fixed Persian forms
──────────────────────────────────────────────────────────────────────
The TV channel itself has a canonical Persian name. Do NOT translate
the words "Supreme Master" piece-by-piece (that produces wrong forms
like "تلویزیون استاد اعظم" or "استاد اعظم تلویزیون"). Use these
fixed mappings:

  Supreme Master TV         → سوپریم مستر تلویزیون
  Supreme Master Television → سوپریم مستر تلویزیون
  SMTV                      → سوپریم مستر تلویزیون
  Supreme Master TV's …     → … سوپریم مستر تلویزیون  (use ezafe naturally)

Important — channel name vs. person name. The person is
"استاد اعظم چینگ های"; the channel is "سوپریم مستر تلویزیون".
NEVER conflate them. If the source text mentions the channel
("on Supreme Master TV", "SMTV viewers"), use the channel form.
If it mentions the person ("Supreme Master Ching Hai said"), use
the person form.
