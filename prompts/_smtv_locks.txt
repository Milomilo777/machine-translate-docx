# PROTECTED TOKEN POLICY (W1–W5)
W1–W4 = frozen byte-id (members of ALLOWED_LATIN). W5 = quoted-title translation policy.
ALLOWED_LATIN = W1–W4 + ACRONYM-IN-PARENS (pattern "Full Name (ACRONYM)" only).

W1) URLs | @handles | #hashtags → BYTE-ID. Do NOT attach sentence-terminal punctuation directly to these spans.
W2) News-source token at end of news line → BYTE-ID and stays at END-OF-LINE.
    (Reuters) (AFP) (AP) (Lao Động) (VTV) (Báo Tin Tức)
    SCOPE: only the source token. Other (...) and [...] translate normally / BRACKETGLOSS.
    Government and descriptive institutional names → translate (US Department of Labor → وزارت کار آمریکا).
W3) TECH_LOCK codes → BYTE-ID
    version (v2.1) | model (GPT-4o, F-16) | formula (H5N1, NaCl) | serial (S01E05)
    VITAMIN CODES: keep the Latin letter byte-id alongside the Persian word
    (ویتامین A | ویتامین D | ویتامین B12 | ویتامین K). This matches Persian
    medical, nutritional, and broadcast convention. Do NOT transliterate the
    letter (✗ ویتامین آ | ✗ ویتامین دی | ✗ ویتامین بی). The letter functions
    as an identifier, not a phoneme. Vitamin codes count as ALLOWED_LATIN
    for this purpose. Reflective check before emit: would a Persian
    pharmacy label show "ویتامین A" or "ویتامین آ"? The label uses Latin.
W4) Placeholders → BYTE-ID
    {…} | %s/%d | $VAR | <span> | \n | \t | path/to/file | 00:01:23,456
    The literal sequence "\n" stays as backslash+n; NEVER rendered as a real newline.
    No sentence-terminal punctuation glued to placeholders.
W5) Quoted titles → TRANSLATE FULLY (not frozen).
    Every word translated/transliterated. No EN residue. Retain " ". Untranslatable fallback → transliterate.
    Title-internal punctuation (?, !) stays INSIDE the quote ("What the Health?" → "سلامتی یعنی چه؟").
    EXCEPTION: VERBATIM/LITERARY quote (attested quotation, scripture, legal text, song line):
      TRANSLATION stage → use established Persian rendering when available; otherwise translate faithfully. NEVER byte-id the English source.
      POLISH stage → Persian verbatim form already in [FA] is locked byte-id by polish stage rule.

# FOREIGN_SCRIPT_AUTHENTIC_VOICE (cross-stage reference)

# SMTV COMPOUNDS — spiritual-equality doctrine
INPUT-DRIVEN ONLY. Activate only when input contains an SMTV compound (or one in ±10 lines WITH COREFERENCE — the plain animal term clearly refers to the same honored animal-person referent(s)). Plain animal words without coreference → ordinary translation. Never invent compounds.

Format: [Honorific]- [Species] (hyphen + ONE space)
✓ شخص- سگ | اشخاص- گاو | شهروند- گربه | نبرد اشخاص- سگ
✗ [S]-[H] | [H]-[S] | [H] - [S]

Honorifics:
  person/being → شخص (sg) | اشخاص (pl)
  citizen/folk/people → شهروند | شهروندان
  friend/companion → دوست
  resident → ساکن | ساکنان
  co-inhabitant → هم‌ساکن (-ان)
  co-citizen → هم‌شهروند (-ان)
  collective/community → اشخاص

Species: from input compound or scan ±10 lines (with coreference gate). Use حیوان only if genuinely unidentifiable.
FORBIDDEN: skip honorific | species in EN | pluralise species | drop hyphen | reorder.

SMTV_OVERRIDE_TRUST: silently auto-correct malformed input compound to canonical format.
