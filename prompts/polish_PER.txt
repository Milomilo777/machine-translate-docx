<MANDATORY_NORMALIZATION>
Hygiene transforms. Apply on ALL lines in PHASE_0_BASE.
MN-1 QUOTE_STYLE — " " ONLY. «» → " ". Sentence-terminal punct OUTSIDE the closing quote. EXCEPTION: punctuation belonging to the quoted title/question/exclamation stays INSIDE ("What the Health?" → "سلامتی یعنی چه؟"). Nested → ASCII '. No new quotes added.
       NO_QUOTE list — NEVER wrap in " ": job titles & honorifics (President, دکتر، استاد), globally famous entities (Apple, Paris, NASA), names immediately after a title anchor.
MN-2 EZAFE_FORM — silent ه → هٔ (خانهٔ، درجهٔ، اتحادیهٔ). Never ه‌ی. ZWNJ for prefixes (می‌، نمی‌)، suffixes (ها، تر، ترین)، established compounds (هم‌خانه، هم‌ساکن، دست‌وپاگیر، به‌شمار، بی‌خانمان). Avoid decorative or post-preposition ZWNJ.
MN-3 HARAKAT — strip all by default. Exceptions:
       1) Established VERBATIM/LITERARY/scriptural forms where attested harakat is essential — fully preserved; NOT counted in ceiling.
       2) Rare unresolved homographs — both required: genuine homograph AND unresolvable by ±5 context. Ceiling/100: NEWS/FACILITATOR ≤3, SAGE ≤5.
MN-4 NO_BAASHE — "باشه"/"باشهٔ" → "بله" / "درست است" / "حتماً". Applies inside ordinary quotes too.
       COLLOQUIAL_NORMALIZE (broadcast-safe): آره→بله | میشه→می‌شود | میرم→می‌روم | خوبه→خوب است. Preserve communicative function.
       FILLERS (persona-aware): Yeah→بله | OK→بسیار خب/خوب | Sure→حتماً | Of course→البته | Thanks→سپاس/متشکرم/ممنون | Sorry→ببخشید/متأسفم.
MN-5 NO_TOOSAT — "توسط" treatment:
       PREDICATE line → active voice within same line; preserve passive only if agent unknown/absent.
       FRAGMENT line → no active conversion; replace "توسط" with prepositional (از سوی / به دست / زیر نظر / با حمایت) by context.
       👑 MASTER line — prefer MINIMAL prepositional repair over active recast when active voice would alter rhetorical emphasis.
       Never invent agent.
MN-6 PUNCT_NORMALIZE — outside protected quotes (LS-10) AND outside number contexts:
       "؛" and ASCII ";" → "،" or clause split within SAME line.
       ASCII "?" → "؟". ASCII "," → "،".
       NUMBER parsing precedes this rule: ASCII comma between digits handled by MN-10, NEVER converted to "،".
       No Oxford comma in LISTS. Comma before "و" IS PERMITTED for independent clauses or to prevent misreading.
       No new brackets, glosses, translator notes added by polisher.
       DASH_NORMALIZE: any — / – / -- variant → spaced ASCII " - ". Never en/em dash.
       Do not glue sentence-terminal punctuation to ALLOWED_LATIN spans.
MN-7 SERIAL — قسمت N از M. قسمت locked.
MN-8 LATIN_LEAKAGE — outside ALLOWED_LATIN → Persianise (AI → هوش مصنوعی | EU → اتحادیهٔ اروپا | DNA → دی‌ان‌ای). Applies inside ordinary quotes.
       LATIN_PHRASES (not TECH_LOCK): per capita → سرانه | de facto → عملاً | status quo → وضع موجود | et al. → و همکاران.
MN-9 ORTHOGRAPHY — Arabic ك → Persian ک. Arabic ي → Persian ی. Remove tatweel (ـ). Use only U+200C for ZWNJ. Does NOT apply inside ALLOWED_LATIN or LS-10 verbatim quotes.
MN-10 NUMBER_FORMAT — apply NUMBERS policy (digit Persianisation, thousands separator "٬", quantity-only rounding, currency/ordinal/fraction). Value-locked.
       Does NOT alter numbers inside LS-10 verbatim quotes or ALLOWED_LATIN spans.
       WRITTEN_NUMBERS — factual written-out numbers → Persian digits when unambiguous (ninety percent → ۹۰٪ | ten people → ۱۰ نفر). Skip for idioms/scripture/titles/Master rhetoric.
       NEVER apply rounding to: years (2026), IDs/codes (F-16, S01E05, H5N1), episode/article/legal numbers, addresses, postcodes, timestamps, model/serial/phone numbers.
       Apply MILLION rule BEFORE THOUSAND. ۳٬۰۰۰→۳ هزار | ۱۵٬۰۰۰→۱۵ هزار | ۱٬۰۰۰→هزار | ۲٬۰۰۰٬۰۰۰→۲ میلیون. Non-round (۱٬۵۰۰/۳٬۷۵۰) stay.
       Date: D Month YYYY / Month D, YYYY → ۴ مارس ۲۰۲۶. Ambiguous numeric (03/04/2026) → Persianize digits, preserve order/separator.
       Month names (locked): January→ژانویه | February→فوریه | March→مارس | April→آوریل | May→مه | June→ژوئن | July→ژوئیه | August→اوت | September→سپتامبر | October→اکتبر | November→نوامبر | December→دسامبر.
       Currency: $5 million → ۵ میلیون دلار. Ordinals: 1st → نخست/اول. Fractions: 1/2 → نیم / یک‌دوم.
</MANDATORY_NORMALIZATION>

<SEMANTIC_ANCHORS>
Run on EVERY line in PHASE_0_BASE (exception to EN_ROLE default). NOT retranslation; anchor mismatch only.
SA-1 NEGATION — affirmative ↔ negative match.
       SCOPE_ATTACHMENT_GUARD: do not let a local attachment turn a clause-level negation/action into a noun-level concept. e.g. "not eating" → "نمی‌خورد" (clause-level), NOT "نه-خوردن"/"روزه‌داری" (noun-level error) unless source clearly intends that nominalization.
SA-2 NUMBERS — digit value, quantity, percent, range match.
SA-3 NAMES — proper nouns transliterated correctly.
SA-4 DATES — D Month YYYY match.
SA-5 SPEAKER + COREFERENCE — who/whom match. Pronoun and ellipsis chains preserved across line context. If Persian grammar forces an explicit referent, choose the least-committal context-consistent form and keep it consistent.
SA-6 MODALITY — distinguish:
       obligation (must/shall) → باید | prohibition (must not) → نباید
       permission (may/can — license) → مجاز است / می‌تواند | ability (can) → می‌تواند
       possibility (may/might/could) → ممکن است / شاید
       logical inference (must be) → حتماً / لابد | recommendation (should) → باید / بهتر است
       counterfactuality (could have / would have) → می‌توانست / ممکن بود
     Do NOT map all may/can/must to one Persian form.
SA-7 CORE_PREDICATE — main verb action match (different action class only). IDIOM-AWARE: if Persian idiom preserves communicative meaning of an EN idiom/metaphor, surface verb mismatch is NOT an anchor error (made my day → دلم را شاد کرد is NOT a SA-7 failure).
SA-8 SPIRITUAL_TERM — sky/heaven/heavenly mapping (آسمان physical / بهشت divine / بهشتی spiritual quality).
SA-9 INTENSITY — source-supported intensifiers preserved; translator-invented certainty/intensity removed/scaled.
SA-10 ASPECT/TIME — already / still / no longer / yet / began / continued / used to / has been match.
SA-11 PROTECTED_SPAN_PRESENCE — ALLOWED_LATIN spans (W1–W4 + ACRONYM-IN-PARENS) present in [EN] must appear byte-id in output, unless line is corrupt. Overridden by BLANK POLICY when [FA] is blank.
SA-12 LOGICAL_CONNECTOR — because / although / despite / unless / so that / in order to / therefore / however: logical relation preserved.
SA-13 COMPARISON_SCOPE — only / even / at least / at most / more than / less than / no more than / not all / all: quantifier scope preserved.
SA-14 ONTOLOGICAL_REPAIR — if [FA] contains an ontological mismatch (inanimate subject + human/religious action, e.g., "اعتکاف یخچال" for "glacier retreats") → reverse-engineer EN source and use the contextual Persian synonym ("عقب‌نشینی یخچال"). NEVER apply to SMTV ontology terms (LS-2, LS-4, LS-8). Use EN as the corrective compass; this is the one case polisher repairs translator-side semantic drift.

SA-12 / SA-13 ANCHOR-ONLY constraint: if the logical/scope relation is already unambiguous in [FA], do NOT insert an explicit connector merely to mirror [EN]. These anchors detect mismatch, not literal token mapping.

On MISMATCH:
  [FA] anchor wrong → fix the anchor only, within same line, within G7.
  Anchor matches but phrasing differs → leave it (HANDOFF_TRUST).
</SEMANTIC_ANCHORS>
